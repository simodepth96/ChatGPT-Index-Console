# OpenAI URL Inspection Tool

This project offers an experimental Streamlit application designed to test whether a given webpage is:

1. **Cached in OpenAI‚Äôs internal index**, or
2. **Fetched live via ChatGPT‚Äôs web search (grounding)**

According to the official [OpenAI Web Search documentation](https://platform.openai.com/docs/guides/tools-web-search#live-internet-access), models can operate either on cached/indexed content or perform a live fetch, depending on the `external_web_access` setting.
By default, **ChatGPT Search uses live web access**, but internal caching appears to exist to avoid repeatedly refetching identical content.

This tool explores whether a URL is included in OpenAI‚Äôs cached index and doesn't need a live search from the web.

---

## Using the App

The [Streamlit app](https://openai-index-console.streamlit.app/) allows you to:

* Provide a URL
* Check if a page exists in OpenAI's cache (default `external_web_access: False`)
* If a URL is not featured, the app infers whether the page:

  * **Exists in OpenAI‚Äôs cached index**
  * Is **blocked by JS / CSR rendering**
  * Requires **fresh grounding** to be interpreted

## Background & Motivation

OpenAI‚Äôs documentation hints at an internal cached index used to speed up or stabilise responses generated by ChatGPT-User. Direct confirmation is scarce, but behaviour suggests:

* OpenAI sometimes **serves pre-indexed HTML snapshots**
* When unavailable, ChatGPT relies on **web search grounding**
* When both fail (e.g., heavy CSR), ChatGPT may:

  * Produce thin, incomplete summaries
  * Rely on **semantic proximity**, similar to TF-IDF retrieval
  * Infer content from *related sites* in the same topical cluster
